                      Will Haley Mod blackpit • 2 years ago

                      Hi blackpit. Sorry for my delayed response. Is the goal to go back and make edits, re-package the img, and then redeploy it? In that case, I think it would make sense to mount the img file...

                      sudo losetup --partscan --show --find debian.img
                      sudo mount /dev/loop0p1 /mnt/debian
                      sudo mount -t proc /proc /mnt/debian/proc
                      sudo mount -t sysfs /sys /mnt/debian/sys
                      sudo mount -o bind /dev /mnt/debian/dev

                      And then chroot to it.

                      sudo chroot /mnt/debian

                      At this point, you can edit the img file as you need. Once you're done, you can unmount everything.

                      sudo umount /mnt/debian/{dev,sys,proc}
                      sudo umount /mnt/debian
                      sudo losetup -d /dev/loop0

                      And then redeploy the img file as you see fit.

                      dd if=debian.img of=/dev/sdz

                      I hope that helps!
